1) Project Vision

NeuroSynth aims to predict neurological decline risk from clinical and biomarker signals, eventually evolving into a multimodal platform that includes imaging, genomics, temporal modeling, causality, and clinical report generation.

The current implementation keeps this vision but delivers a practical, single-developer runnable path.

2) Original Architecture (Enterprise)

Original Designed for production-scale MLOps and cloud operations:

  • Cloud infra contracts and templates in terraform/
  • Kubernetes packaging in helm/
  • Dockerized service path via Dockerfile and docker-compose.yml
  • CI/CD and quality/security gates in .github/workflows/
  • Large source layout in src/neurosynth/ for advanced modules
  • Extensive tests in tests/

3) Current Architecture (Minimal)

Current Lean runnable flow built from your OASIS CSV:

  • explore.py: dataset inspection and sanity checks
  • train.py: RandomForest training and artifact export
  • main.py: FastAPI inference service (/predict)
  • index.html: browser-based local UI
  • app.py: Gradio app for Hugging Face Spaces
  • requirements.txt: simple runtime dependencies

5) What Was Preserved

  • Repository name remains NeuroSynth
  • Enterprise directories remain intact and untouched
  • CI, Docker, Terraform, Helm, and source modules are still available for future growth

6) What Was Simplified

  • Reduced immediate required stack to Python + scikit-learn + FastAPI
  • Focused target to binary classification (Demented vs Nondemented)
  • Moved deployment recommendation to no-cost path first

7) Current End-to-End Workflow

Data

Input CSV is expected at project root: oasis_longitudinal.csv.

Exploration

Run python explore.py to inspect shape, columns, null counts, and class distribution.

Training

Run python train.py to train and produce:

  • model.pkl
  • label_encoder.pkl

API

Run uvicorn main:app --reload and call POST /predict.

UI

Open index.html locally, submit biomarker values, and view prediction/confidence.

Free Deployment

Use app.py in Hugging Face Spaces with requirements.txt.

8) Quality, Risks, and Limits

  • Model is a practical baseline, not a clinical-grade validated decision system
  • Dataset size and label quality constrain generalization
  • No external calibration/monitoring loop in minimal mode
  • Clinical use requires governance, validation, and regulatory controls

9) Step-by-Step Path Back to Advanced NeuroSynth

Phase A (Now to Stable Baseline)

  1. Version artifacts and metrics per training run
  2. Add robust input validation and error handling
  3. Add basic model-card and reproducibility notes

Phase B (Intermediate)

  1. Add experiment tracking (MLflow or similar)
  2. Containerize minimal app and add health checks
  3. Add simple staging deploy pipeline

Phase C (Advanced)

  1. Reintegrate temporal and multimodal modules from src/neurosynth/
  2. Restore cloud infra deployment paths in Terraform/Helm
  3. Enable secret-managed production deploy with strict gates
